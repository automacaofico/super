Como atualizar o Painel Unifilar

Passo a passo para publicar uma planilha nova no painel automacaofico.github.io/super. Leva cerca de 2 minutos e não exige nenhum conhecimento técnico.

⏱ 2 minutos 🔁 Repita toda vez que a planilha for atualizada 🌐 Só precisa de um navegador

Resumo em 3 passos

  1. Abrir o repositório no GitHub
  2. Enviar (upload) a planilha nova, com o nome exatamente igual
  3. Clicar em "Commit changes" para confirmar

Antes de começar

i

O que você vai precisar

Um login no GitHub com acesso ao repositório automacaofico/super, e o arquivo da planilha atualizado no computador, salvo com este nome exato:

Modelo Acompanhamento Super - FINAL.xlsx

⚠️
O nome do arquivo importa. O painel só encontra a planilha se o nome for idêntico a esse, letra por letra (incluindo maiúsculas/minúsculas e espaços). Não renomeie o arquivo.

Enviando a planilha nova

1

Abra o repositório

Acesse este endereço no navegador e faça login no GitHub se for pedido:

github.com/automacaofico/super ↗

github.com/automacaofico/super
＋ Add file ▾ clique aqui
📁 .git
📊 Modelo Acompanhamento Super - FINAL.xlsx
📄 index.html
2

Clique em "Add file" → "Upload files"

No canto superior direito da lista de arquivos, clique no botão Add file e escolha a opção Upload files no menu que aparece.

3

Envie a planilha atualizada

Arraste o arquivo Modelo Acompanhamento Super - FINAL.xlsx para a área pontilhada da página (ou clique nela para escolher o arquivo pelo Explorador de Arquivos).

github.com/automacaofico/super/upload/main
📤 Arraste os arquivos aqui para adicioná-los ao seu repositório
ou escolha os arquivos
💡
Como o nome é igual ao arquivo que já existe no repositório, o GitHub vai simplesmente substituir a versão antiga pela nova — não cria um arquivo duplicado.
4

Confirme o envio ("Commit changes")

Espere a barra de envio terminar (fica verde), role a página para baixo até a seção Commit changes e clique no botão verde para confirmar.

github.com/automacaofico/super/upload/main
Atualização da planilha — 15/07/2026
Commit changes clique aqui

Não precisa mudar nada nos campos — o texto sugerido pelo próprio GitHub já serve. Só clicar no botão verde.

Conferindo se deu certo

5

Abra o painel e confira a barra do topo

Espere cerca de 1 minuto e abra (ou atualize, com F5) o painel público:

automacaofico.github.io/super ↗

automacaofico.github.io/super
Atualizado do GitHub · 15/07/2026, 09:14

Se a data e hora ali baterem com o momento em que você acabou de enviar o arquivo, deu tudo certo — o painel já está mostrando os dados novos para qualquer pessoa que abrir o link.

Pronto. É só isso — sempre que a planilha for atualizada, repita esses 5 passos. Não é preciso mexer em mais nada no site.

Se algo não sair como esperado

O painel abriu com os dados antigos

Espere mais um pouco (às vezes leva até 2 minutos) e atualize a página forçando o recarregamento: no Windows, Ctrl + F5. Se ainda assim não mudar, clique no botão ↻ Atualizar agora na barra no topo do painel.

Apareceu uma mensagem de erro/aviso amarelo no topo do painel

Confira se o nome do arquivo enviado é exatamente Modelo Acompanhamento Super - FINAL.xlsx e se a planilha mantém as abas originais (ENTRADA, MATERIAIS etc.). Se o problema continuar, mande um print da mensagem para quem configurou o painel.

Não tenho acesso para fazer login no GitHub

Peça para o responsável pelo repositório (automacaofico) te adicionar como colaborador. Sem isso, o botão "Add file" não aparece.
